Space Complexity: O(n) — We store one schedule array of size equal to the maximum deadline, plus the sorted jobs list.
Phase breakdown: Sorting n jobs costs O(n log n). For each job we scan backwards through slots — worst case O(n) per job → O(n²) overall. Using a Union-Find structure reduces slot lookup to O(α(n)) ≈ O(1), giving O(n log n) total.

Sorting & Searching
How sorting by profit and slot-search work inside the algorithm.
Why sort first? The greedy algorithm's correctness depends entirely on processing jobs in decreasing profit order. Without sorting, a low-profit job can occupy a slot needed by a higher-profit job, making the result suboptimal.

Greedy Strategy
How the greedy paradigm applies, why it works, and where it fails.
Paradigm: Greedy. At each step, choose the remaining job with the highest profit and assign it to the latest available slot before its deadline. Once a decision is made, it is never reconsidered.
Greedy Choices — Classic Example
Why Does Greedy Work? (Exchange Argument)
Exchange argument: Suppose an optimal solution doesn't schedule the highest-profit feasible job first. We can always swap in that job without decreasing total profit. By repeating this argument for every position, the greedy order produces an optimal schedule.
Counterexample — Earliest Deadline First (Wrong!)
EDF fails here: Sorting by earliest deadline ignores profit entirely. A low-value job with a tight deadline fills a slot that could have held a high-value job, reducing total profit.
Key Concepts

Greedy Choice Property

A globally optimal solution can be assembled from locally optimal (greedy) choices. Here: always pick the highest-profit available job.

Optimal Substructure

After placing one job, the remaining scheduling problem has the same structure and can be solved optimally by the same greedy rule.

Common Mistake

Assigning to the earliest free slot instead of the latest. This wastes scheduling flexibility and leads to suboptimal results.

Feasibility Check

A job is feasible only if at least one slot ≤ its deadline is free. If none exists, the job must be skipped regardless of profit.

Algorithm Overview
What the problem is, how the algorithm works, and a fully worked example.

What is the problem?

▼
Given n jobs, each with a profit pᵢ and a deadline dᵢ (each job takes exactly 1 time unit), schedule a subset to maximise total profit. Each job must complete by its deadline; only one job can run per time slot.

Input: List of (name, profit, deadline) triples.
Output: Subset of jobs + assigned slots, maximising total profit.

How does the algorithm work?

▼
1. Sort all jobs by profit (descending) in O(n log n).
2. Create a slots array of size = max deadline (all empty).
3. For each job (in sorted order):
   • Scan backwards from deadline to slot 1.
   • If a free slot is found → assign the job there.
   • If none found → skip the job.
4. Return the filled slots as the optimal schedule.

When should you use this algorithm?

▼
Use this algorithm when:
• Each task takes exactly one unit of time.
• Each task has a hard deadline.
• You want to maximise total profit / reward.
• Tasks can be skipped if infeasible.

Real-world analogies: CPU job scheduling, task prioritisation, project bidding.
